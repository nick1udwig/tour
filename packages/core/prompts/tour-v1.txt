Critical formatting rules:
- `lines=A-B` must be absolute file line numbers.
- Every line commentary heading must refer to absolute lines inside that snippet's `A..B` range.
- Use `## Line <X> commentary` for a single line.
- Use `## Lines <Y>-<Z> commentary` for a line range.
- `lines=A-B` defines the lines of interest; frontend derives focus indicators from this range.
- Do not provide manual highlight metadata.
- Keep bullets flat (`- item`), no nested bullets.
- Do not use ordered lists for commentary content.
- In commentary, use inline markdown only; do not use fenced code blocks.
- Keep snippets tight (roughly 6-30 lines), high signal only.
- Never add commentary for trivial lines (imports, obvious assignments, straightforward return values).
- Prefer commentary on tricky control flow, invariants, edge cases, state transitions, and cross-module effects.
- Do not emit extra top-level `##` sections beyond `## Overview` and `## Line.../## Lines...`.
- If you need extra structure under overview, use `###` headings.
