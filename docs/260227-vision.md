Codebase tour tool

Currently when you ask an llm for a codebase tour it just gives you a bunch of text which is helpful but not optimal

It is common to give a codebase tour when introducing someone to a new codebase. This looks like screen sharing and walking through the high-level architecture, important data flows, important architectural features, basic operation (building, running, deployment) on GitHub or in an ide

In principle we should be able to create the same thing with ai with tts, showing code lines, highlighting at appropriate times. It could even be interactive: user asks a question and ai stops to answer before continuing on tour. This is the “final product”

The mvp is a webpage that is like a slide deck walking through the code. It has the code under discussion nicely displayed along with highlighting and commentary on relevant parts of the code. Easy links to GitHub for each code snippet. User inputs a GitHub url and gets out slide deck

First interface is a cli like

tour https://github.com/foo/bar

That clones repo, plugs it into codex with prompt, and builds out simple slide deck site (have preexisting styling so codex just has to build the slides, ie output files, code lines, commentary). Serve it at a localhost webpage. If user looks at page before it’s ready show a loading spinner
